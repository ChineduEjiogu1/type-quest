# TypeQuest

A Monkeytype-style typing test with a lightweight RPG progression layer. Type to
earn XP, climb levels and ranks, and unlock longer tests and harder word packs —
all driven by how well you actually type.

## Design philosophy

One rule holds the whole design together: **every RPG number traces back to a
keystroke.** XP, levels, unlocks, and stats are never a separate grind sitting
beside the typing test — they are the typing test's own metrics, re-skinned as
progression. If a feature can't be traced to a typing signal (speed, accuracy,
consistency, streaks), it doesn't belong in the game.

## Features

**Typing test**
- Timed modes (15s / 30s / 60s)
- Live WPM and a running countdown
- Character-by-character feedback with a moving caret
- Results screen: WPM, accuracy, raw WPM, and the correct/incorrect breakdown

**RPG progression**
- XP earned from `wpm × accuracy × duration`
- Levels and rank titles (Hunt & Peck → Touch Typist → … → Keyboard Sage)
- Unlockable durations and word packs (common / punctuation / code), gated by
  both level and demonstrated skill
- A streak multiplier that builds on clean words and resets on an error
- A character sheet of four derived attributes — Speed, Precision, Stamina, Focus
- Optional depth: weakness-weighted word generation and quests drawn from your
  own error data
- Progress persists across sessions

   │  ├─ analysis.ts               # cross-test: attributes + weakness profile
   │  ├─ progression.ts            # xp / levels / ranks / unlocks / quests
   │  └─ words.ts                  # word pools + generateWords()
   │
   ├─ hooks/
   │  └─ usePersistentProgress.ts  # the only place that touches storage
   │
   └─ components/                  # rendering only
      ├─ Header.tsx                # rank, level, XP bar
      ├─ Settings.tsx              # duration + pack chips
      ├─ TypingField.tsx          # word stream + key handling
      ├─ Word.tsx                  # one word, char-by-char coloring
      ├─ StreakMeter.tsx           # live combo multiplier
      ├─ CharacterSheet.tsx        # Speed / Precision / Stamina / Focus

The root config and the four base `src/` files (`main.tsx`, `App.tsx`,
`index.css`, `vite-env.d.ts`) come from the scaffold — don't hand-write them.
The folders under `src/` (`game/`, `hooks/`, `components/`) are the application.

## Getting started

1. Scaffold the project and install dependencies:

   ```
   npm create vite@latest type-quest -- --template react-ts
   cd type-quest
   npm install
   npm install tailwindcss @tailwindcss/vite
   ```

2. Enable Tailwind v4: add the `@tailwindcss/vite` plugin to `vite.config.ts`,
   then put the Tailwind import line at the top of `src/index.css`. (Tailwind v4
   no longer uses `tailwind.config.js` or `postcss.config.js` by default.)

3. Drop the `game/`, `hooks/`, and `components/` folders into `src/`, and point
   `App.tsx` at them.

4. Start the dev server:

   ```
   npm run dev
   ```

## Architecture

- **`game/`** — pure, React-free, and fully testable in plain Node (you can run
  `tally` or `finishTest` with no DOM). State lives in one object; the reducer
  transitions it via a discriminated-union `Action`, with an `assertNever` guard
  so a forgotten case fails to compile rather than silently doing nothing.
- **`hooks/`** — the single home for persistence. Swapping the storage backend
  is a one-file change.
- **`components/`** — presentation only; they read state and dispatch actions.

## Build order

The structure doubles as a roadmap, smallest change first:

1. **Core** — typing test, XP, levels, ranks, unlocks.
2. **Streak** — `reducer.ts` + `StreakMeter.tsx`. No new data; biggest jump in feel.
3. **Character sheet** — `analysis.ts` + `CharacterSheet.tsx`. Requires a rolling
   result history in `SavedProfile` (build that once; the next step reuses it).
4. **Optional depth** — weakness-weighted words and quests. Skippable; the game
   is complete without them.
